2. Table Creation (Data Definition Language - DDL)
The `CREATE TABLE` command defines the structure of your data, including column names, data types, and initial constraints.

Example 1: Simple Table Creation

```sql
CREATE TABLE employee (
    empid INTEGER PRIMARY KEY,
    ename VARCHAR(20),
    contact DECIMAL(10),
    email VARCHAR(30),
    salary FLOAT
);
```

Example 2: Table Creation with Named Constraint

Constraints ensure data integrity. Here, `pk_empid` is given as an explicit name for the Primary Key constraint.

```sql
CREATE TABLE employee (
    empid INTEGER,
    ename VARCHAR(20),
    contact DECIMAL(10),
    email VARCHAR(30),
    salary FLOAT,
    CONSTRAINT pk_empid PRIMARY KEY (empid)
);
```

Foreign Key Constraint

A Foreign Key links a column in one table to the primary key of another table, enforcing referential integrity. In this example, `department_head` in the `department` table must exist as an `empid` in the `employee` table.

```sql
CREATE TABLE department (
    dept_id INTEGER PRIMARY KEY,
    department_name VARCHAR(20) NOT NULL,
    department_head INTEGER,
    CONSTRAINT fk_dhead FOREIGN KEY(department_head) REFERENCES employee(empid)
);
```

3. Altering Table Structure (The `ALTER TABLE` Command)
The `ALTER TABLE` command is used to modify the structure of an existing table after it has been created.

A. Modifying Columns

```sql
--change datatype of a column
alter table employee 
modify column empid integer not null;

--rename column
alter table employee
modify column empid integer not null, 
change ename employee_name varchar(20);

--changing column name and datatype using change command
alter table employee 
change column ename employee_name varchar(20);
```
B. Adding and Dropping Columns

```sql
-- adding a new column
alter table employee
add column salary float not null;
```

C. Adding System/Timestamp Columns

These columns are useful for tracking when a record was created or last updated.

```sql
ALTER TABLE employee
ADD COLUMN inserted_date DATE,
ADD COLUMN inserted_time TIME,
ADD COLUMN inserted_ts TIMESTAMP;
```

5. Data Manipulation (DML)
These commands are used to manage the rows (data) within the tables.

Example: Inserting Data

The values must be provided in the exact order of the columns defined in the table.

```sql
INSERT INTO employee VALUES (
    1, 'John Doe', 9812311221, 'johndoe123@gmail.com', 45000,
    '2025-04-24', '08:30:14', '2025-04-24 08:30:14'
);
```

Example: Retrieving Data

```sql
-- Retrieves all columns and all rows from the employee table.
SELECT * FROM employee; 
```
