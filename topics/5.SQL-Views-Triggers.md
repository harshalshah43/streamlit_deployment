MySQL Advanced SQL Concepts: Views, Triggers, DCL, & TCL  
This note summarizes advanced SQL concepts—Views, Triggers, DCL, and TCL—using the structure and examples from the Employees Sample Database.

---

## **1. Views**  
A View is a virtual table based on a saved SQL query. It simplifies complex queries and restricts data exposure without storing data itself.  
  
**Example: Current Managers View**

This view only shows the names, departments, and tenure of the current department managers.

```sql
-- 1. Create the View
CREATE VIEW current_department_managers AS
SELECT
    e.emp_no,
    e.first_name,
    e.last_name,
    d.dept_name,
    dm.from_date AS manager_since
FROM
    employees AS e
JOIN
    dept_manager AS dm ON e.emp_no = dm.emp_no
JOIN
    departments AS d ON dm.dept_no = d.dept_no
WHERE
    dm.to_date = '9999-01-01';

-- 2. Query the View (use it like a regular table)
SELECT *
FROM current_department_managers
WHERE dept_name = 'Marketing';
```

## **2. Triggers**
A Trigger is a stored procedure that executes automatically when a specific event (`INSERT`, `UPDATE`, or `DELETE`) occurs on a specific table.

**Example 1: Enforcing a Minimum Salary (BEFORE INSERT)**

This trigger checks a new salary (`NEW.salary`) and modifies the data before it is inserted, ensuring it's never below a threshold ($60,000).

```sql
DELIMITER $$

CREATE TRIGGER before_salary_insert
BEFORE INSERT ON salaries
FOR EACH ROW
BEGIN
    -- Check if the new salary (NEW.salary) is less than 60000
    IF NEW.salary < 60000 THEN
        -- If it is, override the new salary to be 60000
        SET NEW.salary = 60000;
    END IF;
END$$

DELIMITER ;

-- Test the Trigger
INSERT INTO salaries (emp_no, salary, from_date, to_date)
VALUES (10001, 50000, CURDATE(), '9999-01-01');
-- Result: The inserted salary will be 60000.
```

**Example 2: Logging Changes to Another Table (AFTER INSERT)**

This trigger executes an `INSERT` command on a separate log table after a successful action on the main table.

Setup: Create a Log Table

```sql
CREATE TABLE department_log (
    log_id INT AUTO_INCREMENT PRIMARY KEY,
    action_type VARCHAR(50) NOT NULL,
    new_dept_no CHAR(4),
    log_timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Trigger Code

DELIMITER $$

CREATE TRIGGER after_department_insert
AFTER INSERT ON departments
FOR EACH ROW
BEGIN
    -- Insert a log entry into the department_log table using NEW.dept_no
    INSERT INTO department_log (action_type, new_dept_no)
    VALUES ('NEW_DEPARTMENT_CREATED', NEW.dept_no);
END$$

DELIMITER ;

-- Test Case
INSERT INTO departments (dept_no, dept_name) VALUES ('d010', 'R&D');
SELECT * FROM department_log ORDER BY log_id DESC LIMIT 1;
```

**Example 3: Preventing Deletion (BEFORE DELETE & External Check)**

This trigger uses `SELECT...INTO` (a conceptual join) to fetch data from the `salaries` table and raises an error (`SIGNAL SQLSTATE`) to abort the `DELETE` if the employee is a high earner.

```sql
DELIMITER $$

CREATE TRIGGER before_employee_delete
BEFORE DELETE ON employees
FOR EACH ROW
BEGIN
    DECLARE current_salary DECIMAL(10, 2);

    -- Find the employee's current salary using OLD.emp_no
    SELECT s.salary INTO current_salary
    FROM salaries AS s
    WHERE s.emp_no = OLD.emp_no
      AND s.to_date = '9999-01-01';

    -- Check if the current salary exceeds the threshold
    IF current_salary > 150000 THEN
        -- Prevent deletion by signaling an error
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'ERROR: Cannot delete an employee with a current salary over $150,000.';
    END IF;
END$$

DELIMITER ;

-- Test Commands
-- (Assumes emp_no 10001 salary is updated to 160000 for the test)
DELETE FROM employees WHERE emp_no = 10001;
-- Result: The DELETE command fails with the error message.
```

## **4. TCL (Transaction Control Language)**
TCL manages transactions, ensuring that a set of operations are treated as a single, atomic unit of work (either all succeed or all fail).

Example:

```
START TRANSACTION;
UPDATE salaries SET salary = 999999 WHERE emp_no = 10001;
-- If a mistake is found:
ROLLBACK;
-- If everything is correct:
-- COMMIT;
```
